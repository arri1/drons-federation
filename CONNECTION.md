# Подключение фронтенда к бэкенду

## Быстрый старт

### 1. Запустите бэкенд
```bash
cd backend
npm install  # если еще не установлено
docker-compose up -d  # запустить PostgreSQL
npm run prisma:migrate  # применить миграции (если нужно)
npm run init:db  # инициализировать тестовые данные (опционально)
npm run dev  # запустить сервер на http://localhost:3001
```

### 2. Запустите фронтенд
```bash
cd frontend
npm install  # если еще не установлено
npm run dev  # запустить на http://localhost:5173
```

### 3. Проверка подключения

Фронтенд автоматически подключится к бэкенду на `http://localhost:3001/api`.

Если бэкенд запущен на другом адресе:
1. Создайте файл `frontend/.env`
2. Добавьте: `VITE_API_URL=http://your-backend-url/api`

## Проверка работы

1. Откройте http://localhost:5173
2. На главной странице должны отображаться:
   - Топ 3 участника (из базы данных)
   - Предстоящие события
   - Последние новости

## Troubleshooting

### CORS ошибки
- Убедитесь, что бэкенд запущен и CORS настроен (уже настроен в `backend/src/index.js`)

### Нет данных
- Проверьте, что бэкенд запущен: http://localhost:3001/health
- Проверьте, что база данных инициализирована: `npm run init:db` в папке backend
- Проверьте консоль браузера на наличие ошибок

### API не отвечает
- Проверьте, что сервер бэкенда работает
- Проверьте, что PostgreSQL запущен: `docker-compose ps` в папке backend
- Проверьте логи бэкенда
